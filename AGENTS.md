# Codex 开发规则

## 项目方向

1. 本项目只做方向一：智能体应用开发（政务智能问答系统）。
2. 不开发方向二：智能问数应用开发（政务瞭望系统）。
3. 不把爬虫、政务情报收集、多源信息采集、智能报告生成作为本项目核心功能。

## 开发原则

1. 不要一开始生成复杂功能。
2. 优先保证最小可运行闭环：用户提问、知识库检索、大模型回答、前端展示、历史记录。
3. 每次修改前先说明计划。
4. 修改后说明改动文件和每个文件的作用。
5. 不要随意删除已有文件。
6. 不要引入与当前阶段无关的大规模重构。

## 技术约束

1. 后端优先使用 FastAPI。
2. 前端优先使用 Vue。
3. 数据库优先使用 MySQL。
4. 向量数据库优先使用 Chroma。
5. 大模型 API 先保留可配置接口，可支持 DeepSeek 或通义千问。
6. Docker 可作为后续增强，不应在最小闭环阶段过度复杂化。

## 安全与配置

1. 不要把 API Key 写死在代码中，应使用 `.env` 配置。
2. 不要把数据库密码、模型密钥、云服务密钥提交到仓库。
3. 后续应提供 `.env.example` 作为配置模板。
4. 用户输入和日志中涉及手机号、身份证号等敏感信息时，应考虑检测与脱敏。

## 当前阶段限制

1. 当前阶段只做项目规划和开发约束。
2. 不创建正式 `backend`、`frontend` 业务代码目录。
3. 不编写正式业务代码。
4. 后续进入开发阶段前，应先确认最小闭环范围和接口设计。
