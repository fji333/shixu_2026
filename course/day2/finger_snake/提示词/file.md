# 第一部分：结构化需求文档（PRD）

## 项目概述
- **项目名称**：GestureSnake 3D（手势控制3D贪吃蛇）
- **平台**：Web网页应用（PC浏览器）
- **核心玩法**：通过摄像头捕捉食指位置，实时控制3D蛇在游戏区域内的移动

## 技术栈
- **前端**：HTML5 + Three.js（3D渲染）+ MediaPipe Hands（手势识别）
- **后端**：Python Flask（或Node.js Express）
- **数据存储**：服务器本地JSON文件
- **通信**：RESTful API

## 视觉风格参考：八方旅人（Octopath Traveler）HD-2D

本项目采用**HD-2D**视觉风格——将复古像素艺术与现代化3D渲染技术融合，营造独特的"像素油画"质感。

### 核心视觉特征

| 特征 | 实现方式 | 游戏内体现 |
|------|----------|-----------|
| **像素基底** | 所有物体采用低分辨率像素纹理（16×16 ~ 32×32） | 蛇身、食物、地面网格均为块状像素风格 |
| **3D体积光影** | Three.js 实时光照 + 法线贴图 | 蛇身投射动态阴影，食物自发光并照亮周围像素地面 |
| **景深模糊** | 后处理 Bokeh/Depth of Field | 游戏区域边缘适度虚化，聚焦中央游玩区域 |
| **戏剧化光照** | 高对比度定向光 + 环境光遮蔽（AO） | 暗角处理，蛇头附近有较强的点光源跟随 |
| **粒子氛围** | Three.js Points/Sprite 粒子系统 | 背景漂浮光尘/雪花，食物附近有上升的光粒 |
| **水面/反光** | 镜面反射 + 折射材质 | 如地图有水域，呈现像素风格的波光反射 |
| **复古色调** | 高饱和度但偏暖/冷的调色板 | 主色：深靛蓝（背景）、金琥珀（食物光）、暗紫红（蛇身） |

### 具体元素设计

**蛇（Snake）**
- 身体由立方体像素块拼接，每节有明显"体素"感
- 材质：哑光像素纹理，边缘有轻微高光勾勒体积
- 特效：移动时身后拖曳淡淡的光尘粒子；加速时全身泛起药片对应颜色的辉光

**食物与道具**
- 能量球：悬浮的像素球体，中心强光源 + 外围像素光环脉动呼吸
- 苹果：低多边形像素风，表面有油画笔触般的色块，落地时溅起微小像素光点
- 药片：胶囊状像素模型，拾取时爆发一圈向外扩散的像素冲击波

**场景与背景**
- 地面：大块的像素方砖拼贴，接缝处有细微的AO阴影
- 边界墙：穿墙不做实体阻挡，但视觉上远处有雾气/暗角渐隐，体现"无限延伸"感
- 背景层：极远景的像素山峦/建筑剪影，配合视差滚动，带有大气透视（远处偏蓝偏淡）

**UI 界面**
- 字体：像素字体（如 "Press Start 2P" 或 "Zpix"）
- 面板：带像素边框的半透明暗色面板，边框有角落装饰纹样
- 血条/得分：像素 heart 图标 + 复古数字计数器，变化时有弹跳缩放动画

### 技术实现要点
- 使用 Three.js 的 `NearestFilter` 纹理过滤保持像素锐利不模糊
- 开启 `toneMapping`（ACESFilmic）增强光影戏剧感
- 后期处理链：Bloom（辉光）→ DOF（景深）→ Vignette（暗角）
- 所有 Sprite/粒子使用像素风格的 PNG，禁用抗锯齿或仅在UI层开启

## 核心机制

### 1. 手势控制
- 使用MediaPipe Hands检测21个手部关键点
- 取食指指尖（landmark 8）坐标
- 摄像头预览区域镜像显示
- 食指指尖坐标实时映射到游戏区域对应位置
- 蛇头平滑跟随目标位置移动（360度无限转向）

### 2. 3D渲染
- 游戏区域：2D俯视角平面（Top-down view）
- 蛇：3D立体模型渲染（有厚度、有立体感的蛇身在平面上爬行）
- 食物与道具：3D模型渲染
- 背景：简洁的3D风格场景

### 3. 碰撞规则
- **边界碰撞**：穿墙（从一侧出去，从对侧进入）
- **自碰撞**：蛇头碰到自身身体 → 游戏结束

### 4. 食物与道具系统
| 类型 | 外观 | 效果 | 生成 |
|------|------|------|------|
| 基础食物（能量球） | 发光球体 | 得分+蛇身变长 | 随机位置 |
| 苹果 | 红色苹果模型 | 回复生命值 | 随机位置 |
| 药片 | 胶囊模型 | 加速效果（持续5秒） | 随机位置 |

### 5. 难度递增
- 蛇的基础速度随得分增加而提升
- 速度曲线：每得50分提升一个速度等级

### 6. 生命值系统
- 蛇有生命值（初始3点）
- 自碰撞扣1点生命 → 生命值归零时游戏结束
- 苹果道具可回复1点生命（上限3点）

## 界面结构

### 开始界面
- 显示游戏标题 + 3D蛇的动画展示
- "开始游戏"按钮

### 主游戏界面
- **左侧（70%宽度）**：游戏区域（3D渲染画布）
- **右侧上方**：摄像头预览窗口（小窗，显示MediaPipe手部追踪骨架）
- **右侧下方**：得分统计区域（当前得分、生命值、速度倍率、当前道具效果）

### 游戏结束界面
- 显示最终得分
- 输入名字文本框
- "提交排行榜"按钮
- "再来一局"按钮
- 返回主界面按钮

### 排行榜界面
- 从服务器加载Top 10排名
- 显示：排名、玩家名、得分、日期

## 用户系统与数据存储

### 核心设计
- **无密码登录**：输入用户名即可登录，首次使用自动创建用户
- **两套数据**：用户档案 + 排行榜记录（分开存储，结构清晰）
- **两个排行维度**：全局Top 10 + 个人历史最高分

### 数据文件结构（服务器本地JSON）

**文件1：`users.json`** — 用户档案
```json
{
  "zhangsan": {
    "username": "zhangsan",
    "created_at": "2025-05-20 14:30:00",
    "games_played": 12,
    "best_score": 1500,
    "total_score": 8900
  },
  "lisi": {
    "username": "lisi", 
    "created_at": "2025-05-21 09:15:30",
    "games_played": 5,
    "best_score": 800,
    "total_score": 3200
  }
}
```

**文件2：`leaderboard.json`** — 全局排行榜记录
```json
[
  {"username": "zhangsan", "score": 1500, "date": "2025-05-20 16:45:00"},
  {"username": "lisi", "score": 800, "date": "2025-05-21 10:20:15"},
  {"username": "zhangsan", "score": 720, "date": "2025-05-20 15:10:30"}
]
```


### 登录与游戏流程

```
打开网页
  │
  ├─ 显示登录框："请输入用户名"
  │
  ├─ 输入用户名 → 点击"进入游戏"
  │     │
  │     ├─ 用户名已存在 → 登录成功，加载用户档案
  │     └─ 用户名不存在 → 自动创建用户，登录成功
  │
  ├─ 进入开始界面（显示用户名 + 个人最高分）
  │
  ├─ 点击"开始游戏" → 进入主游戏界面
  │
  ├─ 游戏结束
  │     │
  │     ├─ 自动提交得分到服务器
  │     ├─ 显示最终得分 + 是否刷新个人记录
  │     ├─ 显示全局排行榜Top 10（高亮当前玩家）
  │     └─ 显示个人历史得分Top 5
  │
  └─ 点击"再来一局" 或 "退出登录"
```

### 关键逻辑说明

**1. 登录/注册（无密码）**
- 前端发送 `POST /api/login {username: "zhangsan"}`
- 后端检查 `users.json`：
  - 存在 → 返回用户信息
  - 不存在 → 新建用户条目，返回创建成功

**2. 提交得分**
- 游戏结束自动发送 `POST /api/score {username: "zhangsan", score: 1200}`
- 后端同时做两件事：
  - 追加记录到 `leaderboard.json`
  - 更新 `users.json` 中该用户的 `games_played`、`best_score`、`total_score`

**3. 排行榜排序**
- 按 `score` 降序排列
- 返回前10条
- 相同分数时，按 `date` 先后排列（先达到者排前）

**4. 游客模式**
- 可选功能，不记录任何成绩到服务器
- 适合临时体验，不强制登录
