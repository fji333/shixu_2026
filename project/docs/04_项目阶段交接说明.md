# 项目阶段交接说明

## 1. 当前项目方向

本项目聚焦政务服务咨询、政策知识库、智能问答和数据统计，建设目标为“政务服务智能问答平台”。

系统面向政策咨询、办事指南、民生服务问答等场景，通过本地政策知识库、向量检索、模板化答复和数据看板，形成可运行的政务咨询辅助闭环。

当前项目不以政务舆情监测、爬虫采集、多源信息采集或智能报告生成为核心功能。

## 4. 当前运行方式

### 后端启动

```powershell
cd backend
.venv\Scripts\activate
uvicorn app.main:app --reload
```

后端默认地址：

```text
http://127.0.0.1:8000
```

Swagger 地址：

```text
http://127.0.0.1:8000/docs
```

### 前端启动

```powershell
cd frontend
npm run dev
```

前端默认地址：

```text
http://127.0.0.1:5173
```

主要页面：

1. `http://127.0.0.1:5173/`
2. `http://127.0.0.1:5173/chat`
3. `http://127.0.0.1:5173/knowledge`
4. `http://127.0.0.1:5173/dashboard`

### 数据库初始化

先确认 `.env` 中的 `DATABASE_URL` 指向本地 MySQL 数据库，并且目标数据库已创建。

```powershell
python scripts/init_db.py
```

该脚本使用 SQLAlchemy `create_all`，只创建缺失表，不删除已有表和已有数据。

### 分类初始化

```powershell
python scripts/seed_categories.py
```

该脚本会插入默认政务分类，已存在则跳过，不会删除已有数据。

### 示例政策导入

```powershell
python scripts/import_knowledge.py
```

该脚本读取 `data/policies/` 下的 `.md` 文件，将每个文件作为一条 `knowledge_docs` 记录导入。已存在同标题文档会跳过。

### 构建向量索引

在完成数据库初始化、分类初始化和示例政策导入后，从项目根目录运行：

```powershell
python scripts/build_vector_index.py
```

该脚本会：

1. 查询 `knowledge_docs` 中 `status=active` 的文档。
2. 切分文本并写入 `knowledge_chunks`。
3. 使用本地 hash embedding 生成向量。
4. 写入本地 Chroma collection：`gov_knowledge_chunks`。

### 项目一键检查

后端启动后，从项目根目录运行：

```powershell
python scripts/check_project.py
```

该脚本会检查后端健康状态、数据库连接、政务分类、知识库、向量索引、大模型预留状态、数据看板和 `/chat` 问答接口。

## 5. 当前注意事项

1. `.env` 不提交 Git。
2. `backend/.venv` 不提交 Git。
3. `frontend/node_modules` 不提交 Git。
4. `frontend/dist` 不提交 Git。
5. `data/vector_db/chroma.sqlite3` 不提交 Git。
6. `data/vector_db` 下 Chroma 生成的 UUID 向量文件夹不提交 Git。
7. 只保留 `data/vector_db/.gitkeep` 作为目录占位。
8. 当前未配置真实 API Key。
9. 当前 `LLM_ENABLE=false`。
10. 大模型功能默认关闭。
11. 当前系统不会真正调用 DeepSeek、通义千问或其他大模型 API。
12. 当前 `/chat` 的 `answer_mode` 默认为 `template_rag`。
13. 当前回答模式默认为本地知识库辅助生成。
14. 后续启用大模型 API 前，应继续保证 API Key 走 `.env` 配置。
15. 不要把大模型 API Key、数据库密码或云服务密钥写死到代码中。
