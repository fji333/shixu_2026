# 功能模块规划

## 模块总览

本项目以“先完成最小可运行闭环，再逐步增强”为原则。基础功能优先保证用户能够提问、系统能够检索政策知识并调用大模型生成回答；增强功能用于提升安全性、可维护性、用户体验和运营能力。

| 模块 | 模块作用 | 主要功能 | 后续开发优先级 | 功能类型 |
| --- | --- | --- | --- | --- |
| 智能政务问答模块 | 面向用户提供自然语言政务咨询入口 | 接收用户问题；展示回答；支持多轮问答的基础记录；给出政策依据摘要 | P0 | 基础功能 |
| 政策知识库模块 | 存放政策文件、办事指南、常见问答等内容 | 政策文本录入；文档切分；知识条目分类；知识来源记录 | P0 | 基础功能 |
| RAG 检索增强模块 | 在大模型回答前检索相关政策内容，提高回答准确性 | 文本向量化；相似度检索；召回政策片段；拼接参考资料；返回引用来源 | P0 | 基础功能 |
| 大模型调用模块 | 统一封装 DeepSeek、通义千问等大模型 API 调用 | 模型接口配置；提示词模板；请求与响应处理；异常处理；调用日志 | P0 | 基础功能 |
| 用户问答历史模块 | 保存用户提问和系统回答，便于追溯与优化 | 历史记录存储；按用户或会话查询；问答详情查看；基础统计 | P1 | 基础功能 |
| 反馈评价模块 | 收集用户对回答质量的评价 | 点赞/点踩；满意度评分；问题反馈；错误答案标记；反馈列表查看 | P1 | 增强功能 |
| 敏感信息检测与脱敏模块 | 降低用户隐私泄露和合规风险 | 检测手机号、身份证号等敏感信息；输入提醒；日志脱敏；展示脱敏 | P1 | 增强功能 |
| 管理员知识库管理模块 | 支持管理员维护政策知识库 | 新增、编辑、删除政策资料；上传文档；触发向量化；查看知识条目状态 | P1 | 增强功能 |
| 前端展示模块 | 提供用户和管理员可操作的 Web 界面 | 问答页面；历史记录页面；反馈入口；管理员知识库管理页面；基础响应式布局 | P0/P1 | 基础功能 + 增强功能 |
| 后端 API 模块 | 为前端和系统内部能力提供统一服务接口 | 用户问答 API；知识库 API；检索 API；模型调用 API；反馈 API；管理 API | P0 | 基础功能 |

## 开发优先级说明

| 优先级 | 含义 |
| --- | --- |
| P0 | 第一阶段必须完成，用于形成最小可运行闭环 |
| P1 | 第二阶段完成，用于提升系统完整度、可维护性和安全性 |
| P2 | 后续增强，可根据时间和验收要求选择实现 |

## 第一阶段最小闭环建议

第一阶段建议只实现以下闭环：

1. 管理员准备少量政策文本或办事指南。
2. 系统将文本切分并写入 Chroma 向量数据库。
3. 用户在前端输入政务问题。
4. 后端调用 RAG 检索相关政策片段。
5. 后端调用大模型生成回答。
6. 前端展示回答和参考来源。
7. 后端记录问答历史。

该闭环完成后，再逐步加入反馈评价、敏感信息检测、知识库后台管理等功能。
