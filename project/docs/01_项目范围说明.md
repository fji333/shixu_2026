# 项目范围说明

## 项目名称

政务智能问答系统

## 项目背景

本项目来自《四川农业大学 23 级实训项目介绍及前期准备》中的方向一：智能体应用开发（政务智能问答系统）。该方向面向政务服务场景，要求围绕 AI 大模型在政策智能解读、民生诉求处理、政务流程咨询等方面的应用，完成从需求分析、系统设计、编码开发到后续部署验收的工程化实践。

政务服务中常见的问题包括政策条文难理解、办事流程分散、群众咨询重复度高、人工答复成本较高等。智能问答系统可以通过政策知识库、RAG 检索增强和大模型生成能力，为用户提供更清晰、更高效、更可追溯的政务咨询服务。

## 为什么选择方向一

本项目只选择方向一，原因如下：

1. 方向一与智能体应用开发、大模型调用、政务问答、政策解读等能力直接相关，适合作为 AI 应用工程化实训项目。
2. 政务智能问答系统业务边界相对清晰，便于先完成最小可运行闭环，再逐步扩展知识库管理、反馈评价、脱敏等增强能力。
3. 方向一更适合使用 FastAPI、Vue、MySQL、Chroma 和大模型 API 组合实现前后端分离的应用系统。
4. 方向一强调政务服务咨询，不以多源数据采集、爬虫、情报收集为核心，有利于控制项目复杂度和合规风险。

## 系统面向的用户

1. 普通群众：查询政策内容、办理条件、办理材料、办理流程等政务服务问题。
2. 企业用户：咨询涉企政策、补贴申报、资质办理、办事流程等信息。
3. 政务工作人员：辅助回答高频咨询问题，提升咨询处理效率。
4. 系统管理员：维护政策知识库、管理问答数据、查看反馈评价结果。

## 核心业务场景

1. 政策智能问答：用户输入自然语言问题，系统结合政策知识库生成回答。
2. 政策内容解读：将政策条文转化为更容易理解的解释，突出适用对象、申请条件和注意事项。
3. 办事流程咨询：围绕政务事项提供办理材料、办理步骤、办理地点、办理时限等说明。
4. 民生诉求初步引导：对教育、就业、社保、医保、住房等常见民生问题提供咨询入口和参考答复。
5. 问答记录留存：保存用户历史提问和系统回答，便于后续查看、优化和统计。
6. 反馈评价闭环：收集用户对回答的满意度和纠错意见，用于优化知识库和提示词。
7. 敏感信息保护：对身份证号、手机号等敏感信息进行检测、提醒或脱敏处理。

## 不做的内容

本项目当前阶段不做以下内容：

1. 不开发方向二“智能问数应用开发（政务瞭望系统）”。
2. 不将政务情报收集系统作为核心功能。
3. 不开发爬虫采集、反爬策略、IP 代理池、请求头伪装等方向二能力。
4. 不做多源政务信息自动化采集、情报清洗、深度分析、智能报告生成等政务瞭望功能。
5. 不在项目准备阶段创建正式 `backend`、`frontend` 业务代码目录。
6. 不在代码中硬编码任何大模型 API Key 或数据库密码，后续统一使用 `.env` 配置。

本项目的核心边界是：围绕政务智能问答系统建立一个可检索、可问答、可反馈、可管理、可逐步扩展的智能体应用。
