# 技术选型说明

## 技术栈总览

| 类型 | 建议技术 | 作用 |
| --- | --- | --- |
| 后端 | FastAPI | 提供政务问答、知识库、检索、模型调用等 API |
| 前端 | Vue | 构建用户问答界面和管理员管理界面 |
| 数据库 | MySQL | 存储用户、问答历史、反馈、知识库元数据等结构化数据 |
| 向量数据库 | Chroma | 存储政策文本向量，支持 RAG 相似度检索 |
| 大模型 API | DeepSeek 或通义千问 | 生成政务问答结果，接口先保持可配置 |
| 部署 | Docker | 后续用于统一运行环境和部署流程 |

## 后端：FastAPI

FastAPI 适合作为本项目后端框架，主要原因如下：

1. 基于 Python，便于与大模型 API、文本处理、向量检索等 AI 生态集成。
2. 支持自动生成 OpenAPI 文档，方便前后端联调和接口验收。
3. 性能较好，开发体验简洁，适合实训项目快速完成最小可运行闭环。
4. 适合按模块组织问答、知识库、RAG、反馈、管理员等 API。

## 前端：Vue

前端优先选择 Vue，主要原因如下：

1. Vue 上手成本相对低，适合实训团队快速开发。
2. 组件化能力适合拆分问答界面、历史记录、反馈组件、知识库管理页面等界面。
3. 与 Vite、Element Plus 等生态结合方便，适合构建中后台和应用型页面。
4. 后续如果团队更熟悉 React，也可以保留 React 作为备选方案，但默认优先 Vue。

## 数据库：MySQL

MySQL 用于存储结构化业务数据，适合保存：

1. 用户信息或会话信息。
2. 用户提问、系统回答、引用来源等问答历史。
3. 用户反馈、评分、纠错意见。
4. 政策知识库文件信息、分类、上传时间、处理状态等元数据。
5. 管理员操作记录等后续扩展数据。

MySQL 成熟稳定、资料丰富，便于团队协作开发和后续部署。

## 向量数据库：Chroma

Chroma 用于存储政策文本切分后的向量表示，并支持相似度检索。它在本项目中的作用包括：

1. 将政策条文、办事指南、常见问答等文本向量化后写入集合。
2. 根据用户问题检索最相关的政策片段。
3. 为大模型回答提供参考资料，降低幻觉风险。
4. 支撑 RAG 检索增强模块的核心能力。

Chroma 部署和使用较轻量，适合实训阶段快速搭建 RAG 原型。

## 大模型 API：DeepSeek 或通义千问

本项目先保留可配置的大模型接口，不在代码中绑定单一厂商。可优先考虑 DeepSeek 或通义千问，原因如下：

1. 都支持通过 API 调用大模型能力。
2. 适合完成政策解读、问答生成、摘要整理等自然语言任务。
3. 可通过统一配置切换模型供应商，便于后续测试不同模型效果。
4. API Key 必须通过 `.env` 等环境变量配置，不能写死在代码中。

## 部署：Docker

Docker 可作为后续增强能力，不在项目准备阶段强制完成。它适合用于：

1. 统一后端、前端、数据库、向量数据库的运行环境。
2. 降低团队成员之间环境差异带来的问题。
3. 方便后续部署到服务器或云平台。
4. 支撑最终验收时的快速启动和交付。

## 当前阶段约束

当前只做项目准备与规划，不创建正式 `backend`、`frontend` 业务代码目录，不编写业务代码。后续进入开发阶段时，应先实现最小可运行闭环，再扩展管理、反馈、安全和部署能力。
